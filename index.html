Mood-first place discovery

Helping people decide where to go without overthinking.

MoodMap turns “how I feel” into context-aware recommendations — so users don’t have to translate emotions into heavy filters like price, distance, and ratings.

Design question

How can we help people decide where to go without decision fatigue?

What makes this different

Starts with mood (human) and applies constraints quietly (system).

01 — Problem

Decision fatigue during place discovery

People don’t struggle because there are “no options.” They struggle because there are too many, and most apps force filter-based thinking instead of mood-based thinking.

What users feel

Overwhelm → endless scrolling → second guessing → “I don’t know.”

What fails today

Ratings lack context. Filters add work. Users can’t convert emotions into search criteria.

Product opportunity

Start with mood, narrow with lightweight context, and support the “unsure moment.”

✦

Product thesis

If we reduce decision effort (mood-first) and reduce the number of options (curated results), users will choose faster and feel more confident.

02 — Research

I validated the decision fatigue problem before designing the solution

I conducted quick research to confirm decision fatigue behaviors, identify what creates uncertainty, and test whether mood-first input would reduce friction.

Research goal

  • Validate decision fatigue (overwhelm, hesitation, abandonment)
  • Understand what makes users feel “stuck”
  • Evaluate if mood-first feels easier than filter-first

Design principle from research

The UI should feel like a guided decision — not a search task.

Key findings (and what they drove)

Finding 1 — Too many options increases uncertainty

Decision support: show fewer, more relevant recommendations.

Finding 2 — Ratings don’t feel situational

Design response: include “Why this fits your mood” explanations.

Finding 3 — Mood is easier than filters

Design response: mood-first cards, with lightweight context after.

Finding 4 — Users drop off at the “unsure moment”

Design response: compare options + “pick for me” loop.

Insight → Decision

Curated results

Reduce cognitive load by limiting options users see at once.

Insight → Decision

Mood-first input

Start from how users feel — the most natural decision entry point.

Insight → Decision

Decision support loop

Give users help when they hesitate: compare or pick-for-me.

03 — Solution

Mood first. Context second. Support when users feel stuck.

MoodMap guides users through a short flow: choose a mood → adjust preferences → view curated recommendations with reasoning.

Key features

  • Lightweight context setup (budget, distance, group size)
  • Curated recommendations (few, relevant options)
  • Reasoning layer (“Why this fits your mood”)
  • Decision support (compare / pick-for-me)
  • Save / Share to reduce repeat work

Why I believe this is successful

Reduces effort

Users don’t translate mood into filters — the system does it for them.

Reduces uncertainty

Explanations + fewer options increase decision confidence.

Increases completion

Decision support removes the biggest drop-off moment: hesitation.

✓

Success definition

A successful MoodMap session ends with a confident choice — not endless browsing.

04 — System view

Primary journey + decision support loop

The primary flow gets users to recommendations quickly. The decision support loop prevents drop-off when users feel uncertain.

Primary user flow

Start → mood → context → recommendations → details → navigate/save/share.

Decision support flow

If unsure → compare options → still unsure → pick for me → navigate.

06 — Impact & learnings

What this case study proves (UX product thinking)

This is not just UI — it’s a decision-making system designed to reduce cognitive load and increase confidence.

Why behind the idea

Mood is the real input.

People decide based on vibe first — filters are support, not the starting point.

Need behind the product

Decision fatigue is a UX problem.

Choice overload creates uncertainty. The product reduces both effort and options.

Why it’s successful

It helps users finish.

Save/share/navigation + decision support prevents “I don’t know” drop-offs.

What I would measure

  • Time-to-choice (open → navigate)
  • Save/share rate (proxy for satisfaction)
  • Drop-off points (recommendations vs details)
  • Decision support usage (compare / pick-for-me)

Next iteration ideas

  • Feedback loop (“not my vibe” → refine results)
  • More mood presets + accessibility improvements
  • Personalization over time (saved places → better picks)
  • Stronger explainability (“why this place” chips)
